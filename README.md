# 🌸 Подорож мрії — Японія

Навчальний проєкт: сайт-візитівка про мрію подорожувати до Японії.

## 📂 Структура проєкту

```
dream-travel/
├── index.html       — Головна сторінка
├── about.html       — Сторінка "Про Японію"
├── contact.html     — Контактна форма
├── styles.css       — Стилі (з адаптивним дизайном)
├── script.js        — Інтерактивні елементи (JS)
└── README.md        — Цей файл
```

## ✨ Що є на сайті

**Структура:**
- 3 повноцінні сторінки з навігацією між ними
- Заголовки (h1–h4), абзаци, нумеровані та маркеровані списки
- Зображення з підписами (figure / figcaption)
- Семантичні теги (header, main, footer, section, nav)

**Дизайн (CSS):**
- Кольорова палітра в японському стилі (рожевий — сакура)
- Адаптивна верстка через медіа-запити (@media)
- Анімації появи елементів (fadeIn)
- Hover-ефекти на кнопках і посиланнях

**Інтерактивність (JavaScript):**
1. **Кнопка-сюрприз** на головній — змінює фон героя й показує привітання японською (5 варіантів по черзі)
2. **Випадаючий список міст** на сторінці "Про Японію" — обираєш місто, з'являється опис
3. **Форма зворотного зв'язку** з валідацією — перевірка обов'язкових полів та коректності email

## 🚀 Як розмістити сайт на GitHub Pages

Крок за кроком:

### 1. Створи акаунт на GitHub
Перейди на [github.com](https://github.com) і зареєструйся (якщо ще немає акаунту).

### 2. Створи новий репозиторій
- Натисни `+` у правому верхньому куті → `New repository`
- Назва репозиторію: наприклад, `dream-travel`
- Постав галочку **Public** (для безкоштовного GitHub Pages обов'язково)
- НЕ ставь галочку "Add a README file" (у нас він уже є)
- Натисни **Create repository**

### 3. Завантаж файли
Найпростіший спосіб — через веб-інтерфейс:
- На сторінці нового репозиторію натисни **uploading an existing file**
- Перетягни усі файли з папки `dream-travel/` (index.html, about.html, contact.html, styles.css, script.js, README.md)
- Внизу натисни **Commit changes**

### 4. Увімкни GitHub Pages
- Перейди в **Settings** репозиторію (вкладка зверху)
- У лівому меню вибери **Pages**
- У розділі **Source** вибери:
  - Branch: `main`
  - Folder: `/ (root)`
- Натисни **Save**

### 5. Отримай посилання
Через 1–2 хвилини угорі сторінки Pages з'явиться повідомлення:
```
Your site is live at https://ТВОЄ-ІМЯ.github.io/dream-travel/
```
Це твоє публічне посилання! Можеш надсилати його будь-кому.

### 🐛 Якщо щось не працює
- Перевір, що файл називається саме `index.html` (з малої літери)
- Перевір, що репозиторій **Public**
- Зачекай 2–5 хвилин після першого Deploy
- Очисти кеш браузера (Ctrl+F5 / Cmd+Shift+R)

## 🎯 Як перевірити сайт локально (до публікації)

Просто двічі клікни на `index.html` — він відкриється у браузері. Усі стилі та JavaScript працюватимуть.

## 📝 Альтернативні хостинги

Якщо GitHub Pages не підійде:
- **Netlify** — перетягни папку `dream-travel` на [app.netlify.com/drop](https://app.netlify.com/drop), отримаєш посилання за 10 секунд
- **000webhost** — реєстрація → завантаження через File Manager → готово
